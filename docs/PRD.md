## 1. 产品概述

### 1.1 产品定位
Odoo 模块依赖分析器是一款专为 Odoo ERP 系统设计的模块依赖关系分析和升级辅助工具，帮助开发者和系统管理员理解模块间的依赖关系，评估升级风险，并提供自动化的升级辅助功能。

### 1.2 目标用户
- Odoo 模块开发者
- Odoo 系统管理员
- Odoo 实施顾问
- 技术团队负责人

### 1.3 核心价值
| 价值点 | 描述 |
|--------|------|
| **可视化理解** | 直观展示复杂的模块依赖关系 |
| **风险评估** | 量化升级影响，降低升级风险 |
| **效率提升** | 自动化检测和修复，减少人工工作 |
| **知识沉淀** | 保存分析历史，便于团队协作 |

## 2. 功能需求

### 2.1 模块扫描与分析

#### 2.1.1 模块扫描
| 功能项 | 描述 | 状态 |
|--------|------|------|
| 路径扫描 | 扫描本地目录下的所有 Odoo 模块 | ✅ |
| ZIP 上传 | 上传 ZIP 压缩包进行分析 | ✅ |
| Manifest 解析 | 解析 `__manifest__.py` 和 `__openerp__.py` | ✅ |
| 多路径支持 | 同时扫描多个模块目录 | ✅ |
| 历史路径 | 记录并快速选择历史扫描路径 | ✅ |

#### 2.1.2 依赖分析
| 功能项 | 描述 | 状态 |
|--------|------|------|
| 依赖图构建 | 使用 NetworkX 构建有向依赖图 | ✅ |
| 循环依赖检测 | 检测模块间的循环依赖 | ✅ |
| 缺失依赖检测 | 识别未安装的依赖模块 | ✅ |
| 反向依赖查询 | 查询哪些模块依赖指定模块 | ✅ |
| 安装顺序计算 | 计算正确的模块安装顺序 | ✅ |

#### 2.1.3 统计信息
| 统计项 | 描述 |
|--------|------|
| 模块总数 | 扫描到的模块数量 |

### 2.4 升级工具

#### 2.4.1 代码问题检测
| 功能项 | 描述 | 状态 |
|--------|------|------|
| 废弃 API 检测 | 检测各版本废弃的 API 调用 | ✅ |
| 代码模式检测 | 检测旧代码模式（如 @api.multi） | ✅ |
| 自动修复标记 | 标记可自动修复的问题 | ✅ |
| 修复建议 | 提供具体的修复建议 | ✅ |

**支持检测的废弃 API：**
- Odoo 14 → 15: `topological_sort`, `report_sxw`
- Odoo 15 → 16: `pycompat`, `@api.multi`, `@api.one`
- Odoo 16 → 17: `fields.Serialized`, `web.assets_backend`
- Odoo 17 → 18: `oldname` 参数

**支持检测的代码模式：**
- `@api.multi` / `@api.one` 装饰器
- `self.pool.get()` 旧式 ORM
- `from openerp import` 旧式导入
- `browse(cr, uid, ...)` 旧式 API

#### 2.4.2 迁移脚本生成
| 功能项 | 描述 | 状态 |
|--------|------|------|
| pre-migrate.py | 升级前执行的脚本模板 | ✅ |
| post-migrate.py | 升级后执行的脚本模板 | ✅ |
| end-migrate.py | 所有模块升级完成后执行的脚本模板 | ✅ |
| 版本号自动更新 | 自动计算新版本号 | ✅ |
| 保存到模块目录 | 自动保存到 migrations 目录 | ✅ |

#### 2.4.3 升级检查清单
| 分类 | 检查项 |
|------|--------|
| **备份** | 数据库完整备份、代码仓库备份、配置文件备份 |

| **代码** | 代码问题修复、Manifest 版本更新、迁移脚本准备 |

## 3. 非功能需求

### 3.1 性能要求
| 指标 | 要求 |
|------|------|
| 扫描速度 | 500+ 模块 < 5 秒 |

| 层级 | 技术 |

### 4.3 API 接口

#### 扫描相关
| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/api/scan` | 扫描本地路径 |
| POST | `/api/upload` | 上传 ZIP 分析 |

#### 分析相关
| 方法 | 路径 | 描述 |
|------|------|------|
| GET | `/api/graph-data` | 获取依赖图数据 |
| GET | `/api/tree/<module>` | 获取依赖树 |
| GET | `/api/order` | 获取安装顺序 |
| GET | `/api/models` | 获取模型分析 |
| GET | `/api/impact/<module>` | 获取升级影响 |
| POST | `/api/compare` | 版本对比 |

#### 迁移相关
| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/api/migration/analyze` | 分析代码问题 |
| GET | `/api/migration/scripts/<module>` | 获取迁移脚本 |
| POST | `/api/migration/scripts/<module>/save` | 保存迁移脚本 |
| POST | `/api/migration/auto-fix` | 自动修复 |

#### 存储相关
| 方法 | 路径 | 描述 |
|------|------|------|
| POST | `/api/storage/upload` | 上传并保存 |
| GET | `/api/storage/records` | 获取历史记录 |
| POST | `/api/storage/save-current` | 保存当前分析 |
| DELETE | `/api/storage/record/<id>` | 删除记录 |
